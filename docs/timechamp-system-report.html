What Is This System?

We have built a multi-tenant workforce intelligence SaaS platform — a full clone of timechamp.io. The original product charges $3.90–$13.90 per user per month (₹363–₹1,294). Our system targets the Indian SMB market at ₹30 per user per month, making it 12–43× cheaper while delivering identical functionality.

Simple definition: A company installs our desktop agent on every employee's computer. The agent silently records what app each employee is using, takes a screenshot every 5 minutes, tracks location via GPS, and syncs all data to the cloud. Managers log into our web dashboard to see live activity, productivity scores, project progress, and receive alerts when employees are idle or miss clock-in.

Step-by-Step: How One Screenshot Reaches a Manager

1
Agent captures screen (Windows/macOS/Linux) — using platform API (Win32 on Windows, screencapture CLI on macOS, scrot on Linux). The raw image is in memory.
2
Agent resizes + compresses — shared resize.go scales to max 1280px wide using nearest-neighbour algorithm, then JPEG-encodes at Quality 60. Result: ~100KB (was 300KB).
3
Agent saves to local SQLite buffer — stores file path + timestamp locally. If internet is down, nothing is lost. Buffer holds up to 7 days of data.
4
Agent calls POST /agent/sync/screenshot — API authenticates via agent token, generates a Cloudflare R2 presigned PUT URL (valid 5 minutes), and returns it.
5
Agent uploads directly to R2 — the 100KB JPEG goes straight to Cloudflare's edge, bypassing your API server entirely. Your server bandwidth = 0 for screenshots.
6
API saves metadata row to PostgreSQL — only 150 bytes: {id, userId, orgId, s3Key, capturedAt, fileSizeBytes}. The actual image is in R2.
7
Manager opens the screenshot gallery in browser — API queries PostgreSQL for screenshot metadata, generates presigned GET URLs (valid 1 hour), returns them.
8
Browser loads images directly from Cloudflare R2 — served from Cloudflare's CDN edge nodes (200+ locations worldwide). Fast, reliable, zero egress cost for you.
9
Nightly at 2 AM: cleanup cron runs — finds all screenshot rows older than 90 days, deletes each R2 object, deletes the DB row. Keeps storage costs flat forever.

How PostgreSQL Row Level Security (RLS) Protects Every Tenant

Why this matters: Without RLS, if your API has a bug (missing WHERE clause, wrong variable), employee data from Company A could leak to Company B. With RLS, even a bug at the application level cannot return data from another organization — the database itself enforces isolation.

Every table has a policy like this:

CREATE POLICY org_isolation ON activity_events USING (organization_id = current_setting('app.org_id', TRUE)::uuid); -- Before every query, our API sets: SET app.org_id = 'the-org-uuid-from-jwt' -- Now even this buggy query is safe: SELECT * FROM activity_events; -- Result: ONLY returns rows for the current org, enforced by PostgreSQL

Tech Stack — Every Choice Explained

Go (Golang)
Desktop Agent

The agent that runs on every employee's computer captures screenshots, tracks the active window, measures idle time, and syncs data to the API.

Why Go? Compiles to a single static binary (~8MB) with zero runtime dependencies. No Java/Node runtime needed on the employee's machine. Cross-compiles to Windows/macOS/Linux from one codebase using build tags. Starts in <50ms. RAM usage ~15MB idle. Alternatives like Python would need a 100MB+ runtime installed.
NestJS 10
Backend API

The API server handles all authentication, data processing, business logic for 11 modules. Runs on Node.js with TypeScript.

Why NestJS? Opinionated structure (modules, guards, interceptors) prevents spaghetti code in a large codebase. Built-in dependency injection, Swagger docs auto-generated, @nestjs/schedule for cron jobs. TypeScript catches bugs at compile time. Alternatives: FastAPI (Python) or Spring Boot (Java) would require different language expertise and are harder to integrate with the Next.js frontend.
Next.js 14 (App Router)
Web Dashboard

The manager and employee dashboard. 10 pages with role-based UI, real-time polling, charts, and forms.

Why Next.js? Server-side rendering for fast initial load, file-based routing, built-in API routes, excellent TypeScript support. TanStack Query v5 handles all API state (loading, error, caching, refetch). Alternatives: plain React (no SSR), Vue (smaller ecosystem), Angular (too verbose for rapid development).
PostgreSQL + TypeORM
Primary Database

Stores all structured data: users, organizations, attendance, time entries, activity events, screenshot metadata, projects, GPS locations, alerts.

Why PostgreSQL? Row Level Security (RLS) is the critical feature — no other free database has it. This enables true tenant isolation at the database engine level. TypeORM provides typed entities and auto-migration. Alternatives: MySQL (no RLS), MongoDB (no relations, poor for reporting), SQLite (not suitable for multi-tenant production).
Neon (Serverless Postgres)
DB Hosting

Hosts our PostgreSQL database. Auto-pauses when no queries are running (saving money during nights/weekends).

Why Neon? Auto-pause means $0 compute cost during idle hours — critical for a startup with few users. Storage dropped from $1.75 to $0.35/GB-month (April 2026). Free tier includes 100 CU-hours + 0.5GB — enough for the first 50–70 users. Alternatives: AWS RDS ($16/month minimum even idle), Supabase ($25/month Pro for commercial use).
Cloudflare R2
Screenshot Storage

Stores all screenshot JPEGs. S3-compatible API — no code change needed to switch from AWS S3.

Why R2? Zero egress cost — this is the key difference. AWS S3 charges $0.09/GB for downloads. With 100 employees and managers viewing screenshots daily, AWS egress alone would be $10–30/month. R2 egress = $0 forever. Storage is $0.015/GB (vs AWS $0.023/GB). Free 10GB + 10M reads + 1M writes monthly.
Upstash Redis
Caching Layer

Caches live status (30s TTL), app usage analytics (5min/24h TTL), and productivity scores to avoid repeated database joins.

Why Upstash? Serverless (pay per request, not per hour). Free tier: 500,000 commands/month — enough for 100–200 users. No always-on cost. At 100 users, Redis cost = $0. Alternatives: Redis Cloud ($7/month minimum), ElastiCache ($13/month minimum), self-hosted Redis on Hetzner (free but needs management).
Hetzner Cloud
Compute (VPS)

Runs the NestJS API and Next.js web server. CPX21 for early stage (3 vCPU/4GB), CPX31 for scaling (4 vCPU/8GB).

Why Hetzner? 5–10× cheaper than AWS for equivalent compute. CPX21 = $9.5/month vs AWS t3.small = $17/month. Based in Germany/Finland with excellent uptime SLA. Real-world migration data: teams cut costs 89% moving from AWS to Hetzner. Limitation: fewer managed services, but we use serverless alternatives (Neon, Upstash, R2) for those.

Production Reliability — How the System Stays Up

Agent: Works Even When Internet Is Down

The Go agent does NOT send data directly. It first writes everything to a local SQLite database on the employee's machine (the buffer). Then separately, every 30 seconds, it tries to flush the buffer to the API.

Example scenario: Employee loses internet from 10 AM to 11 AM (60 minutes). During this time, the agent captures 12 screenshots + 6 activity batches → all saved to local SQLite. At 11 AM when internet comes back, the agent flushes everything to the API in one sync. Zero data loss. Buffer retains up to 7 days of data locally.

API: Stateless + Horizontally Scalable

The NestJS API is completely stateless — no user sessions stored in API memory. All state lives in PostgreSQL and Redis. This means:

Nightly Maintenance: Data Stays Clean

Every night at 2:00 AM (server time), the MaintenanceService cron job runs:

1
Query PostgreSQL for all screenshots where capturedAt < now() - 90 days
2
For each result: call DeleteObjectCommand on Cloudflare R2 to remove the image file
3
Delete the database metadata row
4
If any single deletion fails (R2 network error), log the error and continue — other screenshots still get cleaned
5
Processes max 500 screenshots per run to avoid memory spikes (batched)
Effect: Storage costs stay FLAT regardless of how long you've been running. A company with 100 employees always has at most 90 days × 100 employees × 9.6MB/day = ~86GB in R2 storage, never more. Cost: $1.29/month forever.
